🛠️ WRITE 框架：高质量文档 5 步法

系统化方法论，让 AI 每次都输出专业级文档

W

What — 明确文档的目标和类型

告诉 AI 你要什么类型的文档：是 JSDoc 注释、API 参考、还是用户指南？目标越清晰，输出越精准。

示例 Prompt
我需要你为这个模块生成 JSDoc 格式的代码注释，
覆盖所有导出函数的参数、返回值和异常情况。
R

Readers — 确定读者是谁

新人入职文档、团队内部 Wiki、还是开源社区 README？读者不同，文档的深度和风格完全不同。

示例 Prompt
读者是刚接触本项目的新同事，假设他们了解 React
但不熟悉我们的业务领域，请用通俗的语言解释。
I

Idiom — 遵循文档惯用格式

每种文档有自己的惯例：JSDoc 有 @param/@returns，Commit 有 Conventional Commits，API 有 OpenAPI。告诉 AI 用什么标准。

示例 Prompt
请遵循 Conventional Commits 规范：
type(scope): description
其中 type 为 feat/fix/docs/refactor 之一。
T

Template — 提供模板或范例给 AI

给 AI 一个你期望的输出模板，它会严格按照格式生成。模板比描述更直观、更不容易出错。

示例 Prompt
请按以下模板格式生成：
## [函数名]
**功能**：一句话描述
**参数**：表格形式
**返回值**：类型 + 描述
**示例**：代码块
E

Examples — 给 AI 展示好的文档示例

Few-shot learning：给 AI 一两个你满意的文档示例，它就能学会你的风格和质量标准。

示例 Prompt
参考以下已有的文档风格来写新函数的注释：

✅ Do — 推荐做法

  • 给 AI 提供函数签名和上下文

❌ Don't — 避免做法

  • 忽略文档的一致性检查
  • 跳过对生成内容的事实核查
  • 用 AI 文档替代架构设计文档
