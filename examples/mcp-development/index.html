🔌 什么是 MCP？

MCP (Model Context Protocol) 是一个开放标准协议，让 AI 模型能够安全地连接和调用外部工具、数据源和服务。它定义了 AI 与工具之间的通信方式，就像 USB 协议让各种设备能接入电脑一样。

😩 传统方式：手动复制粘贴

1 向 AI 描述你需要查询的数据
2 AI 给出 SQL 语句，你手动复制到数据库客户端
3 执行查询，复制结果粘贴回 AI 对话
4 AI 分析结果，给出下一步建议
5 重复以上步骤... 效率低，容易出错

🚀 MCP 方式：AI 直接调用

1 向 AI 描述你需要查询的数据
2 AI 自动通过 MCP 调用数据库工具查询
3 结果直接返回 AI，自动分析并给出结论

搜索文件

search_files
// Tool: search_files — 按关键词搜索文件内容 { "name": "search_files", "description": "在目录中搜索包含关键词的文件", "inputSchema": { "type": "object", "properties": { "directory": { "type": "string" }, "pattern": { "type": "string" } }, "required": ["directory", "pattern"] } } // Handler 实现 async function handleSearchFiles(args: { directory: string; pattern: string; }) { const results = await glob(`${args.directory}/**/*`); const matches = []; for (const file of results) { const content = await fs.readFile(file, 'utf-8'); if (content.includes(args.pattern)) { matches.push(file); } } return { content: [{ type: 'text', text: matches.join('\n') }] }; }

📋 MCP 开发速查表

遵循这些原则，构建安全可靠的 MCP Server。

✅ 推荐做法

  • 为每个工具编写清晰的 description
  • 使用 JSON Schema 验证输入参数
  • 实现优雅的错误处理和错误消息
  • 添加超时和重试机制
  • 使用 TypeScript 增强类型安全

❌ 避免做法

  • 在工具中执行危险的系统操作
  • 忽略输入验证直接处理参数
  • 返回未经处理的原始错误信息
  • 让单个工具承担过多职责
  • 跳过安全性审查直接上线
