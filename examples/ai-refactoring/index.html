提取函数 (Extract Function)

将长函数中的逻辑块提取为独立函数，每个函数只做一件事。

❌ 重构前
function processOrder(order) {
  // 验证订单
  if (!order.items || order.items.length === 0) {
    throw new Error('订单不能为空')
  }
  if (!order.userId) {
    throw new Error('缺少用户ID')
  }
  // 计算总价
  let total = 0
  for (const item of order.items) {
    total += item.price * item.qty
  }
  if (order.coupon) {
    total *= 0.9
  }
  // 创建记录
  return { id: genId(), total, status: 'pending' }
}
✅ 重构后
function validateOrder(order) {
  if (!order.items?.length)
    throw new Error('订单不能为空')
  if (!order.userId)
    throw new Error('缺少用户ID')
}

function calcTotal(items, coupon) {
  const subtotal = items.reduce(
    (sum, i) => sum + i.price * i.qty, 0
  )
  return coupon ? subtotal * 0.9 : subtotal
}

function processOrder(order) {
  validateOrder(order)
  const total = calcTotal(order.items, order.coupon)
  return { id: genId(), total, status: 'pending' }
}
🤖 AI Prompt
将这个 processOrder 函数拆分为多个小函数，每个函数只负责一件事（验证、计算、创建记录）。保持原有行为不变。

消除重复 (Remove Duplication)

合并相似代码为共享函数，避免散弹式修改。

❌ 重构前
function formatUserName(user) {
  const name = user.name.trim()
  return name.charAt(0).toUpperCase()
    + name.slice(1).toLowerCase()
}

function formatCityName(city) {
  const name = city.name.trim()
  return name.charAt(0).toUpperCase()
    + name.slice(1).toLowerCase()
}

function formatProductTitle(product) {
  const name = product.title.trim()
  return name.charAt(0).toUpperCase()
    + name.slice(1).toLowerCase()
}
✅ 重构后
function capitalize(str: string): string {
  const s = str.trim()
  return s.charAt(0).toUpperCase()
    + s.slice(1).toLowerCase()
}

const formatUserName    = (u) => capitalize(u.name)
const formatCityName    = (c) => capitalize(c.name)
const formatProductTitle = (p) => capitalize(p.title)
🤖 AI Prompt
找出这三个函数中的重复逻辑，提取为一个通用的工具函数，然后让原函数调用它。

简化条件 (Simplify Conditionals)

用 early return / 策略模式替代嵌套 if-else，降低认知复杂度。

❌ 重构前
function getDiscount(user) {
  let discount = 0
  if (user) {
    if (user.isVip) {
      if (user.years > 5) {
        discount = 0.3
      } else if (user.years > 2) {
        discount = 0.2
      } else {
        discount = 0.1
      }
    } else {
      if (user.orders > 10) {
        discount = 0.05
      }
    }
  }
  return discount
}
✅ 重构后
function getDiscount(user) {
  if (!user) return 0

  if (!user.isVip) {
    return user.orders > 10 ? 0.05 : 0
  }

  if (user.years > 5) return 0.3
  if (user.years > 2) return 0.2
  return 0.1
}
🤖 AI Prompt
用 early return 模式重构这个嵌套 if-else，减少缩进层级，让逻辑更清晰。保持所有分支的返回值不变。

优化类型 (Improve Types)

用 TypeScript 类型替代 any、增加类型守卫，让编译器帮你捕获错误。

❌ 重构前
function handleEvent(event: any) {
  if (event.type === 'click') {
    doClick(event.x, event.y)
  } else if (event.type === 'key') {
    doKey(event.key)
  } else {
    doOther(event.data)
  }
}

function getConfig(): any {
  return { retries: 3, timeout: 5000 }
}
✅ 重构后
type AppEvent =
  | { type: 'click'; x: number; y: number }
  | { type: 'key';   key: string }
  | { type: 'other'; data: unknown }

function handleEvent(event: AppEvent) {
  switch (event.type) {
    case 'click': return doClick(event.x, event.y)
    case 'key':   return doKey(event.key)
    case 'other': return doOther(event.data)
  }
}

interface AppConfig {
  retries: number
  timeout: number
}
function getConfig(): AppConfig {
  return { retries: 3, timeout: 5000 }
}
🤖 AI Prompt
消除这段代码中所有的 any 类型。为 event 创建可辨识联合类型（discriminated union），为 config 创建接口。确保类型安全。

重命名优化 (Rename for Clarity)

让变量名、函数名更具表达力，代码即文档。

❌ 重构前
function proc(d, f) {
  const r = []
  for (let i = 0; i < d.length; i++) {
    const tmp = f(d[i])
    if (tmp !== null) {
      r.push(tmp)
    }
  }
  return r
}

const x = proc(users, (u) => {
  return u.age >= 18 ? u.name : null
})
✅ 重构后
function filterMap<T, R>(
  items: T[],
  transform: (item: T) => R | null
): R[] {
  const results: R[] = []
  for (const item of items) {
    const transformed = transform(item)
    if (transformed !== null) {
      results.push(transformed)
    }
  }
  return results
}

const adultNames = filterMap(users, (user) =>
  user.age >= 18 ? user.name : null
)
🤖 AI Prompt
重命名这段代码中所有不清晰的变量名和函数名，使其能清晰表达意图。用英文命名，遵循 camelCase 规范。添加泛型类型。

拆分模块 (Split Module)

将大文件按职责拆分为独立模块，每个模块关注单一领域。

❌ 重构前 — utils.ts (500+ 行)
// utils.ts — 什么都往里塞
export function formatDate(...) { ... }
export function parseDate(...) { ... }
export function validateEmail(...) { ... }
export function validatePhone(...) { ... }
export function slugify(...) { ... }
export function truncate(...) { ... }
export function debounce(...) { ... }
export function throttle(...) { ... }
export function deepClone(...) { ... }
export function mergeObjects(...) { ... }
// ... 还有 30 多个函数
✅ 重构后 — 按职责拆分
// utils/date.ts
export { formatDate, parseDate }

// utils/validation.ts
export { validateEmail, validatePhone }

// utils/string.ts
export { slugify, truncate }

// utils/async.ts
export { debounce, throttle }

// utils/object.ts
export { deepClone, mergeObjects }

// utils/index.ts — 统一导出
export * from './date'
export * from './validation'
export * from './string'
export * from './async'
export * from './object'
🤖 AI Prompt
将 utils.ts 按职责拆分为独立模块（date、validation、string、async、object）。创建 index.ts 统一导出，确保现有 import 路径仍然兼容。

🛠️ CLEAN 框架：系统化重构方法

按照 C → L → E → A → N 五步走，让重构有章可循、有迹可查。

C
Consolidate · 合并

找出重复逻辑并整合

扫描代码库，识别相似的函数、重复的模式，将它们合并为共享的工具函数。

扫描这个文件，找出所有重复或相似的代码片段，列出每处重复的位置和可以合并的建议方案。
L
Lift · 提升

提取通用逻辑到更高层级

将嵌套在组件/模块内部的通用逻辑提升为可复用的公共模块。

找出这些组件中重复使用的逻辑，将其提取为自定义 Hook 或公共模块，放到 utils/ 或 hooks/ 目录下。
E
Extract · 提取

将复杂函数拆分为小函数

超过 20 行的函数考虑拆分，每个函数只负责一个清晰的职责。

这个函数超过 50 行且包含多个职责。将其拆分为多个小函数，每个函数名清晰表达其职责。保持原有行为不变。
A
Align · 对齐

统一代码风格和命名规范

确保整个代码库使用一致的命名风格、错误处理模式和代码组织方式。

审查这个目录下的所有文件，找出命名风格不一致的地方（如混用 camelCase 和 snake_case），并统一为 camelCase。
N
Name · 命名

用清晰的名称传达意图

好的命名胜过注释。让每个变量名、函数名、类名都能立刻传达其用途。

审查这段代码中所有变量和函数的命名。找出含糊或误导性的名称，给出更能表达意图的替代名称和重命名理由。
