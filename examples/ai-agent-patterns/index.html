🤖 什么是 AI Agent？

从"问答机器人"到"全能助手"的进化

AI Agent 是一个能自主感知环境、制定计划、调用工具并执行多步任务的 AI 系统。它不只是回答问题，而是主动完成目标。

普通 AI

问答机器人

你问一句它答一句。没有记忆、不会规划、无法使用工具。每次对话都是独立的一问一答。

AI Agent

全能助手

你说目标，它自己规划方案、调用工具、完成任务。能记住上下文，遇到问题会自主调整策略。

核心组件

🧠

规划能力

将复杂任务分解为可执行步骤

🔧

工具调用

调用 API、读写文件、执行代码

💾

记忆系统

维护上下文和中间状态

🔄

反馈循环

观察结果、调整策略、持续迭代

自动化工作流

端到端自动化：从需求到部署全流程自主执行

硬编码的 if-else 流程，需要预定义所有路径。遇到新情况只能报错退出。

// 传统自动化脚本 — 刚性流程
function deploy(config) {
  if (config.type === 'frontend') {
    runBuild()
    uploadToS3()
  } else if (config.type === 'backend') {
    runTests()
    buildDocker()
    pushToRegistry()
  } else {
    throw new Error('未知部署类型!')
    // 无法处理新情况
  }
}

🛠️ PLAN 框架：构建 Agent 的 4 步法

一个简洁有效的 Agent 开发框架

P

Perceive — 感知

获取环境信息

获取环境信息和任务上下文。读取用户输入、检查系统状态、收集可用资源。

L

Learn — 学习

理解与策略

理解目标、识别可用工具、制定执行策略。将任务拆分为可执行的子步骤。

A

Act — 行动

执行操作

执行工具调用、生成代码、与外部系统交互。按计划逐步完成每个子任务。

N

Navigate — 导航

评估与决策

评估结果、处理错误、决定下一步。如果失败，调整策略重新尝试。

4 种 Agent 设计模式

1

单 Agent 模式

▼

一个 Agent 处理所有任务，适合简单场景。Agent 独立完成感知、推理、行动的全过程。

用户输入 → Agent → 工具调用 → 输出结果
2

链式模式

▼

Agent 串行执行，前一个的输出是后一个的输入。适合流水线式任务处理。

Agent A → Agent B → Agent C → 最终结果
3

路由模式

▼

分类器 Agent 将任务分发给专用 Agent。每个专用 Agent 负责特定领域的处理。

输入 → 路由 Agent → 专用 A
输入 → 路由 → 专用 B
4

协作模式

▼

多个 Agent 并行工作，互相协调。通过共享状态和消息传递实现协同。

代码 Agent ⟷ 协调器 ⟷ 测试 Agent
占位 ⟷ 协调 ⟷ 文档 Agent

✅ 推荐做法

✅限制 Agent 的最大迭代次数
✅为每个工具添加超时控制
✅使用结构化输出规范 Agent 回复
✅记录完整的执行轨迹
✅实现优雅的降级策略

❌ 避免做法

❌让 Agent 无限循环执行
❌给 Agent 过多不相关的工具
❌忽略中间步骤的错误处理
❌信任 Agent 的所有输出不做验证
❌跳过安全边界限制
