❌ 传统分支切换

1 git stash — 暂存当前改动

Review Worktree

不离开当前工作，创建独立目录审查 PR

▼

当你正在开发功能，同事提交了一个 PR 需要审查。无需 stash，直接创建一个新的 worktree：

$ git worktree add ../review-pr-123 origin/pr-123 # 审查完毕后清理 $ git worktree remove ../review-pr-123

💡 审查完成后记得清理 worktree，保持项目整洁。

🧪

Experiment Worktree

大胆实验新方案，不影响主线开发

▼

想尝试一个激进的重构方案？用 worktree 隔离实验，失败了直接删除，零风险：

$ git worktree add ../experiment feature/experiment # 实验成功 → 合并回主分支 # 实验失败 → 直接删除 $ git worktree remove ../experiment

💡 实验分支独立存在，不会污染你的功能分支。

🚨

Hotfix Worktree

紧急修复线上问题，功能开发不中断

▼

生产环境出了 bug！从 main 分支创建 worktree，立即修复，同时你的功能分支保持原样：

$ git worktree add ../hotfix-urgent main $ cd ../hotfix-urgent # 修复 bug，提交，推送 $ git push origin main $ cd ../project $ git worktree remove ../hotfix-urgent

💡 关键时刻零延迟响应，这就是 worktree 的价值。

🤖

AI Agent Worktree

让 Claude Code 在隔离环境中独立工作

▼

Claude Code 内置 worktree 支持，AI 在独立目录中工作，不影响你的主工作区：

# Claude Code 中使用 > EnterWorktree # 创建隔离 worktree > ...执行任务... # AI 独立工作 > ExitWorktree(keep) # 保留 worktree > ExitWorktree(remove) # 或删除 worktree

💡 多个 AI Agent 可在不同 worktree 中并行工作。

命令速查表

日常使用最频繁的 Worktree 命令

git worktree add <path> <branch>

创建新的 worktree。将指定分支检出到一个新目录中，两个目录共享同一个 .git 仓库。

git worktree list

列出当前仓库的所有 worktree，包括路径、HEAD 和分支信息。定期检查，避免遗忘。

git worktree remove <path>

移除指定的 worktree 目录。如果有未提交的改动，需要加 --force 标志。

git worktree prune

清理已手动删除目录但未正式移除的 worktree 记录。保持 .git/worktrees 干净。

git worktree add --detach <path> <commit>

以 detached HEAD 方式创建 worktree，适合临时查看某个特定 commit 的状态。

git worktree lock <path>

锁定 worktree，防止被 prune 误删。适合存储在可移动设备上的 worktree。

🤖 Claude Code 深度集成

Claude Code 原生支持 Git Worktree，为 AI 辅助开发提供完美的隔离环境。

🔀 EnterWorktree

在 .claude/worktrees/ 中创建新的 worktree 和分支，AI 在隔离环境中工作。

🚪 ExitWorktree

退出 worktree 会话。选择 keep 保留代码，或 remove 清理环境。

🛡️ 完美隔离

AI 修改不会影响主工作区，多个 Agent 可以并行处理不同任务。

⚡ 自动清理

会话结束时自动提示是否保留或删除 worktree，避免资源浪费。

📋 典型工作流示例

# 1. 用户在主工作区开发功能 you: 帮我重构 auth 模块，用 worktree 隔离工作 # 2. Claude Code 创建隔离 worktree claude: EnterWorktree(name: "refactor-auth") # → 创建 .claude/worktrees/refactor-auth/ # → 新分支 claude/refactor-auth 基于 HEAD # 3. AI 在隔离环境中工作 claude: 修改文件、运行测试、提交代码… # 4. 工作完成，退出并保留代码 claude: ExitWorktree(action: "keep") # → 回到主工作目录 # → worktree 代码保留供审查 # 5. 用户审查后合并或丢弃 you: git merge claude/refactor-auth

最佳实践

掌握这些准则，用好 Worktree

✅ 推荐做法

  • 用 worktree 做代码审查，保持主工作区干净
  • 使用有意义的 worktree 目录名（如 review-pr-123、hotfix-auth）
  • 任务完成后及时清理，运行 git worktree list 定期检查
  • 利用 worktree 实现 CI/CD 隔离构建
  • 配合 AI Agent 使用，实现真正的并行开发
  • 团队共享 worktree 命名规范，便于协作

❌ 避免事项

  • 不要同时维护超过 5 个 worktree，管理成本会急剧上升
  • 不要忘记清理废弃的 worktree，用 git worktree prune 清扫
  • 不要在多个 worktree 中修改同一个文件，容易产生合并冲突
  • 不要为了一行代码的小改动创建 worktree，杀鸡不用牛刀
  • 不要在 worktree 中直接修改 .git 目录，它们是共享的
  • 不要手动删除 worktree 目录，应使用 git worktree remove
